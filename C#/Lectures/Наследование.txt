1. abstract class

abstract class: приблизетельно равно interface, который может содержать обычные виртуальные и
 абстрактные методы поля и свойства

2. interface – это способ декларирования методов свойств полей с последующей реализацией в производном классе

3. class
{
	virtual Methods

}

Обычные классы могут содержать виртуальные и/или обычные методы, свойства поля


4. В чем отличие между абсткными классами и интерфесами

если интерфейсы могут содержать только описание (или декларирование), которое будет реализовано в производных классах

то абстракные класссы могут содержать в себе виртуальные функции и обычные методы. В абстракных классах также могут быть абстрактные
методы, которые необходимо реализовать исключительно в производных классах.
